A. Scripting 
Scripting adalah kegiatan menulis serangkaian perintah untuk dieksekusi oleh shell. Scripting dapat menggabungkan perintah yang ingin dilakukan pada bash yang panjang dan berulang dari ke dalam satu script line function call. Hasil Scripting yang disimpan dapat mempermudah user untuk record perintah yang pernah dilakukan dan recalling perintah yang sudah dilakukan. 
Bash Linux tersedia di hampir semua varian Linux dan tidak memerlukan instalasi terpisah, untuk memulai Scripting dapat menggunakan command "script namafile.txt" namafile.txt adalah file baru yang akan menyimpan hasil dari script tersebut. hasil dari script memuat :
1. Tanggal dan jam dari "Script started"
2. Tanggal dan jam dari "Script done"
3. Tanggal dan jam dari "Prompt"
Untuk menyelesaikan script dapat menggunakan command "exit" dan hasil script sudah tersimpan pada file baru tadi.


B. FUSE 
FUSE adalah singkatan dari Filesystem in Userspace merupakan suatu software interface pada UNIX. Fitur paling penting dari FUSE adalah FUSE memperbolehkan non-privileged user untuk melakukan mount dengan aman. Biasanya FUSE digunakan untuk membuat virtual file system.  Untuk membuat FUSE ada 3 hal yang harus dibuat, yaitu kernel side module, libfuse, dan aplikasinya
Untuk mengimplementasikan file system, kita harus menulis program terlebih dahulu yang akan berfungsi untuk melakukan mount pada file system yang baru, merespon jika ada operasi read/write/stat dan program ini juga akan akan tertaut dengan libfuse. Ketika file system berhasil di mount, handler akan teregister dengan kernel dan meneruskan operasi read/write/stat kepada handler untuk dilakukan respon terhadap operasi tersebut.


C. Git
Git adalah version control system yang digunakan para developer untuk mengembangkan software secara bersama-bersama. Fungsi utama git yaitu mengatur versi dari source code program Anda dengan mengasih tanda baris dan code mana yang ditambah atau diganti.
Perintah-perintah dasar git:
Git init : untuk membuat repository pada file lokal yang nantinya ada folder .git
Git status : untuk mengetahui status dari repository lokal
Git add : menambahkan file baru pada repository yang dipilih
Git commit : untuk menyimpan perubahan yang dilakukan, tetapi tidak ada perubahan pada remote repository.
Git push : untuk mengirimkan perubahan file setelah di commit ke remote repository.
Git branch : melihat seluruh branch yang ada pada repository
Git checkout : menukar branch yang aktif dengan branchyang dipilih
Git merge : untuk menggabungkan branch yang aktif dan branch yang dipilih
Git clone : membuat Salinan repository lokal


D. GitHub
GitHub adalah sebuah aplikasi berbasis website dengan Version Control System (VCS) yang menyediakan layanan untuk menyimpan repository dengan gratis. Repository merupakan tempat yang dapat Anda gunakan untuk menyimpan berbagai file berupa source code. Namun tak hanya itu, dengan aplikasi ini Anda juga dapat melakukan kolaborasi dalam mengerjakan project tertentu agar dapat terus melakukan update secara rutin, termasuk melacak dan menyimpan perubahan-perubahan yang terjadi dalam project tersebut. Aplikasi ini termasuk sangat populer dan banyak digunakan termasuk oleh perusahaan-perusahaan besar skala gunia seperti Facebook, Google, dan Twitter.

Github juga memiliki kelebihan berupa Github Pages. Github Pages adalah fasilitas hosting web statis (html, css, javascript) yang diberikan secara gratis dengan subdomain .github.io. Github Pages sangat berguna untuk  membuat dokumentasi misalnya untuk suatu repo karena Github Pages dapat mempublikasikan website dari repo kita dengan mudah dan gratis Github Pages.

GnuPG
GnuPG adalah suatu software yang dipakai untuk melakukan proses enkripsi data, dekripsi data , dan penandatanganan digital untuk selanjutnya dilakukan proses pertukaran data antar perangkat. GnuPG juga dikenal sebagai GPG yaitu suatu command line yang dapat dijalankan melalui aplikasi Terminal di setiap sistem operasi dengan fitur easy integration dengan aplikasi lainnya. Enkripsi dengan GPG ini sudah mengimplementasikan standar yang dijelaskan dalam RFC4880 (dikenal juga dengan standar PGP). GPG juga dilengkapi dengan fungsi manajemen kunci keamanan seperti pembuatan kunci, penyuntingan kunci, dan penghapusan kunci keamanan.
Dengan model enkripsi yang digunakan pada GPG, seseorang pasti mempunyai sepasang kunci keamanan yaitu Private Key dan Public Key. Dimana Private Key adalah kunci enkripsi yang hanya boleh diakses oleh pemilik kunci, sedangkan public key sebisa mungkin harus disebarkan seluas-luasnya.


F. SHA1SUM
sha1sum adalah program komputer yang menghitung dan memverifikasi hash SHA-1. Ini biasanya digunakan untuk memverifikasi integritas file. Ini memungkinkan dibandingkan dengan intisari pesan yang diterbitkan untuk memeriksa apakah file tersebut tidak dimodifikasi dari aslinya. Pada UAS OS genap 2020, file SHA1SUM digunakan sebagai berkas yang menyimpan hash berkas-berkas yang diupload di GitHub. SHA-1 menghasilkan nilai hash 160-bit (20 byte) yang dikenal sebagai intisari pesan. sha1sum di instal secara default di sebagian besar distribusi Linux. Varian termasuk shasum (yang memungkinkan SHA-1 hingga SHA-512 fungsi hash untuk dipilih secara manual), sha224sum, sha256sum, sha384sum dan sha512sum, yang menggunakan fungsi hash SHA-2 tertentu, dan sha3sum (yang memungkinkan SHA-3 melalui SHA3- 512, SHAKE, RawSHAKE, dan Keccak berfungsi untuk dipilih secara manual).


G. SHA1SUM.asc
Format .asc sendiri merupakan format untuk berkas ASCII lapis baja yang digunakan oleh suatu software yang dipakai untuk melakukan proses enkripsi data seperti PGP atau GnuPG. File tersebut berisi pesan yang ditandatangani secara digital dan dapat menyimpan informasi tertulis teks biasa, serta informasi biner yang disandikan sebagai teks. File ASC juga menyertakan kunci sebagai teks yang ditandatangani jelas, yang dapat diverifikasi menggunakan metode otentikasi. Oleh karena itu SHA1SUM.asc merupakan berkas dimana berisi sebuah hasil enkripsi SHA1SUM yang nantinya dapat diverifikasi dan didekripsi (oleh yang berhak) untuk melihat informasi asli yang ada.

F. GNUPG
Penggunaan versi gpg yang berbeda membuat peserta bingung. Terdapat perbedaan fungsi apabila menggunakan versi gpg yang berbeda sehingga peserta hanya mengikuti instruksi yang berikan tanpa mengerti perbedaan fungsi dari versi gpg yang digunakan. Untuk gpg2 sepertinya tidak melihat kunci pribadi yang gpg melihatnya. Misalnya, gpg2 --list-secret-keys tidak memberikan output, tetapi gpg --list-secret-keys memberikan output. Beberapa peserta mengalami kesalahan pada berkas 0000-mypub.txt, salah satunya karena peserta membuat berkas secara langsung di github dan copy paste isi berkas dari badak sehingga ada beberapa bagian yang terpotong dan tidak terbaca oleh sistem. Selain itu, beberapa peserta memiliki key lebih dari 1 key dan mencantumkan lebih dari 1 key pada berkas 0000-mypub.txt , sedangkan yang perintah hanya menginginkan 1 key dan harus key tersebut yang akan dipakai. Kesalahan lainnya yaitu terdapat secret key pada 0000-mypub.txt karena secret key hanya boleh diakses oleh pemilik kunci, sedangkan berkas perlu dibaca oleh dosen. Gpg ini dapat digunakan untuk enkripsi baik dengan atau tanpa tanda tangan, sehingga mahasiswa perlu lebih teliti membaca perintah yang diberikan untuk membedakan berkas yang perlu ditandatangani juga atau hanya enkripsi saja. Pada pelakanaan UAS OS, peserta juga mendapatkan pemahaman baru mengenai clear sign yaitu peserta dapat menandatangani berkas dalam bentuk clear text.


G. SHA1SUM dan SHA1SUM.asc
Mayoritas peserta baru mengenal dan mengetahui tentang sha1sum dan sha1sum.asc pada saat pelaksanaan UAS OS. sha1sum merupakan berkas yang menyimpan hash berkas-berkas yang diupload di GitHub. sha1sum.asc merupakan berkas dimana berisi sebuah hasil enkripsi SHA1SUM yang nantinya dapat diverifikasi dan didekripsi (oleh yang berhak) untuk melihat informasi asli yang ada. sebelum pelaksanaan UAS OS, dosen memberikan arahan untuk memahami mengenai gpg dan sha1sum, serta memberikan contoh penggunaan keduanya melalui forum diskusi SCELE. Namun, mayoritas peserta hanya mengikuti contoh tanpa memahami terlebih dahulu penggunaan gpg dan sha1sum. sha1sum dan sha1sum.asc perlu dibuat ulang setiap ada perubahan. Beberapa peserta mengalami kesalahan pada sha1sum dan sha1sum.asc karena kurang mengerti kegunaan keduanya sehingga tidak membuat ulang setiap ada perubahan. 


H. Race Condition
Race condition terjadi awalnya pada saat absen yang dilakukan di forum SCELE untuk kebutuhan absen tiap minggunya selama masa pembelajaran jarak jauh. Kemudian, beberapa peserta memberi saran untuk absen melalui github. Awalnya absen cukup berjalan lancar karena waktu untuk absen cukup panjang sehingga peserta tidak terburu-buru untuk absen secara berbarengan. Namun, race condition tetap terjadi terutama saat absen untuk pelaksanaan UAS OS karena waktu absen yang cukup sempit.


I. Absen
Pada absen mingguan selama masa pembelajaran jarak jauh, beberapa peserta absen terlambat karena tidak mengetahui batasan waktu terakhir absen dan tidak melihat SCELE secara berkala. Selain itu, beberapa peserta menganggap lebih baik absen lebih lambat untuk menghindari race condition.


J. SCELE
Frekuensi peserta melihat SCELE sangat jarang karena adanya WhatsApp group sehingga peserta cenderung membuka SCELE apabila ada pemberitahuan melalui WhatsApp group. Peserta biasanya akan melihat SCELE apabila ingin absen dan melihat materi pada waktu pembelajaran di mana terjadi 2 kali seminggu, selain waktu pembelajaran peserta cenderung tidak melihat SCELE kecuali ada informasi penting yang disampaikan melalui WhatsApp Group.Terlalu banyak thread pada forum di scele terkadang membuat peserta kebingungan untuk mencari suatu informasi pada forum. Forum SCELE cukup baik penggunaannya untuk diskusi dan tanya jawab karena tidak akan hilang informasinya kecuali dihapus, serta pembahasan untuk kasus yang berbeda dapat dibuat dengan thread yang berbeda.


K. WhatsApp group
Penggunaan WhatsApp group cukup efektif untuk memberitahu terkait informasi penting. Hal ini dikarenakan WhatsApp group memberikan notifikasi apabila terdapat pesan masuk, sedangkan SCELE tidak memberikan notifikasi. Peserta cenderung cepat tanggap terhadap informasi apabila terdapat notifikasi. WhatsApp group kurang baik penggunaannya untuk diskusi dan tanya jawab karena pembahasan akan terus tertimbun apabila ada pesan masuk.


L. Zoom
Penggunaan zoom pada pelaksanaan mata kuliah OS genap 2020 hanya beberapa kali untuk sesi tanya jawab. Beberapa peserta cenderung lebih suka apabila materi atau demo disampaikan secara langsung melalui zoom. Hal ini dikarenakan mahasiswa dapat bertanya secara langsung apabila terdapat hal yang membingungkan dan dapat terjawab sampai jelas.


M. Youtube
Materi yang disampaikan mudah diterima, namun kurang banyak contoh demo yang diberikan. Contoh demo akan sangat berguna untuk pemahaman saat pelaksanaan UAS seperti pada mata kuliah OS genap 2020. Selain itu, peserta dapat mengulang kembali video apabila ingin memahami lebih dalam kapan saja.


N. Email
Penggunaan email sudah tepat untuk permasalahan penting yang bukan terkait materi pembelajaran. Penggunaan subjek [HELP] juga penting untuk menunjukkan bahwa email tersebut penting dan dapat membedakan dengan email lainnya.


O. Scripting
Scripting berfungsi untuk menyimpan sekumpulan instruksi pemrograman yang ditafsirkan pada saat runtime. Pada penggunaan di mata kuliah OS ini, hasil scripting juga dapat menjadi alat pengecekan bahwa peserta mengerjakan secara mandiri dan pada waktu yang benar. Namun, mayoritas peserta tidak memahami maksud dari langkah-langkah scripting sebelumnya sehingga peserta hanya mengikuti langkah-langkah yang diberikan dengan cara copy dan paste.
